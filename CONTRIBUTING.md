# Contributing

## Branching
- Create a branch from `main` for each task.
- Branch naming:
  - `feature/...` for features
  - `docs/...` for documentation
  - `chore/...` for repository maintenance

## Commits
Use Conventional Commits:
- `feat: ...`
- `docs: ...`
- `chore: ...`

## Pull Requests
- Open a PR into `main` for every change (including docs).
- Keep PRs small and focused.
- Add a short description of what was done and how to test/verify.
